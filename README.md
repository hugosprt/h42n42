# H42N42

H42N42 is a browser game written in OCaml with Eliom and `js_of_ocaml`. The page is generated by the Eliom application, the gameplay runs on the client side, and the project is served by `ocsigenserver`.

## Project goal

The player must keep a population of Creets alive in a contaminated world:

- a toxic river at the top infects healthy Creets on contact
- infected Creets can contaminate other healthy Creets
- the hospital at the bottom heals only sick Creets manually dropped there by the player
- the game gets harder over time as movement speed increases
- the game ends when no healthy Creet remains alive

## Requirements

- Docker
- Docker Compose plugin (`docker compose`) or legacy `docker-compose`

## Repository structure

- `src/main.eliom`: Eliom application entrypoint and client game logic
- `static/style.css`: interface and game styling
- `h42n42.conf.in`: Ocsigen and Eliom server configuration
- `Makefile`: build automation with `all`, `clean`, `fclean`, `re`
- `Dockerfile`: container build and dependency installation
- `docker-compose.yml`: one-command launch for evaluation

## Run with Docker

```bash
docker compose up --build
```

or:

```bash
docker-compose up --build
```

Then open:

- [http://localhost:8080/](http://localhost:8080/)

## Local build commands

```bash
make all
make clean
make fclean
make re
```

`make all` builds:

- the server module in `_server/h42n42.cma`
- the client bundle in `static/h42n42.js`

## Implemented mandatory features

- HTML page generated by Eliom and TyXML
- client-side OCaml compiled to JavaScript with `js_of_ocaml`
- mouse interactions handled with `Lwt_js_events`
- one movement Lwt thread per Creet
- Creets displayed as dynamic HTML elements
- linear movement with random direction changes
- reflection on game area boundaries
- contamination on river contact
- contamination spread by distance/radius collision checks
- sick Creets move slower than healthy ones
- drag-and-drop interaction for moving Creets
- dragged Creets are temporarily immune to contamination
- healing only when a regular sick Creet is manually dropped in the hospital
- berserk Creets grow over time and die when they become too large
- mean Creets shrink, chase healthy Creets, and die after one minute
- periodic spawning while at least one healthy Creet exists
- progressive difficulty through global speed increase
- visible `GAME OVER` overlay when the healthy population is gone

## Notes

- The project is a real Eliom application served from `/`, not a static `index.html`.
- Static assets are served from `static/`.
- `docker compose up --build` builds the application inside the image and starts the Ocsigen server on port `8080`.
